# テレアポ管理シート × Telegram (Google Apps Script版)

n8n も Docker も不要。**Googleスプレッドシートに付属するスクリプトだけで完結**します。
担当者がTelegramボットに「3 アポ獲得」と送ると、シートが自動更新されます。

## 必要なもの (5分で揃う)

1. **Googleアカウント** (もうある)
2. **テレアポ管理シート** (Googleスプレッドシートで開いた状態)
3. **Telegramアプリ** (スマホ or PC)

これだけ。サーバーもクレジットカードも不要。

---

## 手順 (全部で7ステップ・約10分)

### ① Telegramボットを作る (1分)

1. Telegramで `@BotFather` を検索 → トーク開始
2. `/newbot` と送信
3. 「ボットの名前」と「ユーザー名」を聞かれる → 適当に答える (例: `MyCallLogBot`、ユーザー名は `mycalllog_bot` のように `_bot` で終わる)
4. **Bot Token** が表示される (例: `123456789:AAH...XYZ`) → コピー

### ② スプレッドシートを開く

`テレアポ管理シート` を Googleスプレッドシートで開く。
(まだGoogleにアップしてなければ、`テレアポ管理シート.csv` を Googleドライブにアップロード → 「Googleスプレッドシートとして開く」)

### ③ Apps Scriptエディタを開く

メニューの **拡張機能 → Apps Script** をクリック。
別タブでスクリプトエディタが開く。

### ④ コードを貼り付ける

1. デフォルトの `function myFunction() {}` を全部削除
2. `apps-script-call-log/Code.gs` の中身を全部コピー → 貼り付け
3. ファイル名は `Code.gs` のままでOK

### ⑤ Bot Tokenを書き換える

ファイル冒頭の:
```js
const BOT_TOKEN  = 'ここにBotFatherでもらったTokenを貼る';
```
を、①でコピーしたTokenに書き換える。

`SHEET_NAME` がシートのタブ名 (`テレアポ管理シート`) と一致しているかも確認。

### ⑥ ウェブアプリとしてデプロイ

1. 右上の **デプロイ → 新しいデプロイ** をクリック
2. 歯車アイコン → 種類を **ウェブアプリ** に設定
3. 設定:
   - **次のユーザーとして実行**: 自分
   - **アクセスできるユーザー**: **全員** ⚠️ (これ重要 / Telegramからアクセスできるように)
4. **デプロイ** → 権限を求められたら「許可」
5. **ウェブアプリのURL** が表示される (例: `https://script.google.com/macros/s/AKfycby.../exec`) → コピー

### ⑦ TelegramにWebhook登録

1. スクリプトエディタに戻る
2. `registerWebhook` 関数の中の:
   ```js
   const WEBHOOK_URL = 'ここにデプロイで発行されたウェブアプリURLを貼る';
   ```
   を⑥でコピーしたURLに書き換える
3. 上部の関数選択ドロップダウンで **registerWebhook** を選択 → **実行** ボタン
4. ログに `{"ok":true,"result":true,...}` と出れば成功 ✨

## 使い方

Telegramで作ったボットを開いて、以下のフォーマットで送信:

```
3 アポ獲得      ← No.3 にアポ獲得を記録
12 不在         ← No.12 に不在を記録
/log 7 再架電   ← /log プレフィックスもOK
5:番号違い      ← コロン区切りもOK
```

ボットから `✅ 記録しました...` と返事が来れば成功。
シートを見ると、該当行の **架電日N / 架電結果N / 架電ステータス** が更新されています。

## 結果キーワード対応表

| 入力 | 設定される 架電ステータス | Slack通知 |
| --- | --- | --- |
| アポ獲得 / 獲得 / 成約 | アポ獲得 | ✅ |
| アポ | アポ予定 | — |
| 不在 / 留守 | 次回架電予定 | — |
| 拒否 / 断り / NG | 対応不可 | — |
| 再架電 / 折り返し | 次回架電予定 | — |
| 番号違い / 誤番 | 番号誤り | — |
| その他 | 架電済み | — |

## (任意) Slack通知

アポ獲得時にSlackチャンネルに通知したい場合:

1. Slackで [Incoming Webhook](https://api.slack.com/messaging/webhooks) を作成 → URL取得
2. `Code.gs` の `SLACK_WEBHOOK_URL` に貼り付け
3. 再度 **デプロイ → 新しいデプロイ** で更新

## トラブルシューティング

| 症状 | 対処 |
| --- | --- |
| ボットが反応しない | `setup` 関数を実行して Token が正しいか確認 |
| `No. が見つかりません` | シート1行目のヘッダーが `No.` (英ピリオド) になっているか確認 |
| `シート「テレアポ管理シート」が見つかりません` | タブ名と `SHEET_NAME` が一致しているか確認 |
| 権限エラー | デプロイ時に「アクセスできるユーザー: 全員」になっているか確認 |
| Webhook登録に失敗 | `WEBHOOK_URL` の末尾が `/exec` になっているか確認 |

## n8n版との違い

| 項目 | Apps Script版 (このフォルダ) | n8n版 (`../n8n-workflows/`) |
| --- | --- | --- |
| ホスティング | 不要 (Google管理) | 必要 (n8n Cloud / Railway等) |
| 料金 | 完全無料 | n8n Cloud $20/月〜 など |
| セットアップ難易度 | ★★☆ コピペ7ステップ | ★★★ 認証情報3つ登録 |
| 拡張性 | コード修正が必要 | GUIでノード追加 |
| おすすめ | **個人/小規模チーム** | チーム運用・他連携が多い |
