# 税務 — 不足資料回収トラッカー

第 8 期決算（決算月 2026/7 末 → 8 月末申告）に向けた、税理士提出書類の裏付け資料（契約書・請求書等）の**回収進捗トラッカー**です。

## 目的

`不足資料調査/`（マーシャルアーツ様＿不足資料リスト、2026-05-27 時点）で棚卸しした不足資料のうち、**高額・優先案件**の回収ステータスを一元管理し、決算確定までの残タスクを可視化する。

| ファイル | 内容 |
| --- | --- |
| `2026-06-01_不足資料回収トラッカー.csv` | 優先案件 33 件（入金 19 / 出金 14）の回収ステータス。UTF-8 (BOM) / 13 列。 |

> **データ出所:** `不足資料調査/README.md` §5（高額・優先案件）。**データ基準日 2026-05-27**、トラッカー作成日 2026-06-01。全件（入金約120件・出金約55件）ではなく高額サブセットを対象。

## 列定義

| 列 | 列名 | 内容 |
| --- | --- | --- |
| 1 | `区分` | `入金` / `出金` |
| 2 | `優先度` | 回収優先度（本リストは全件 `高`） |
| 3 | `取引先` | 取引先名 |
| 4 | `対象日` | 入金日 / 出金日（複数・多数の場合はその旨） |
| 5 | `金額` | 金額（円）。`多数`案件は概算（備考参照） |
| 6 | `不足種別` | 不足している資料の種別 |
| 7 | `Drive状況` | Drive 内の有無（`あり` / `なし`） |
| 8 | `Slack状況` | Slack ヒット状況（`あり` / `なし` / `未確認` / `-` / `確認できず`） |
| 9 | `回収ステータス` | 下記凡例参照 |
| 10 | `次アクション` | 次に取るべき作業 |
| 11 | `担当` | 担当者（未割当） |
| 12 | `期限` | 対応期限（未設定） |
| 13 | `備考` | 補足 |

## 回収ステータス凡例

| ステータス | 意味 |
| --- | --- |
| `回収済` | 必要資料が Drive 内に揃っている |
| `要確認` | 資料はあるが金額・物件・対応関係のギャップ確認が必要 |
| `移行待ち` | Slack 上にあり、Drive への移行が必要（ファイル ID は次アクション欄に記載） |
| `要取得` | Drive/Slack いずれにも未発見。先方等から別途取得が必要 |

## 運用

- 進捗があった行の `回収ステータス` / `担当` / `期限` / `備考` を更新する。
- 全件版・詳細は `不足資料調査/README.md` を参照。重複削除・フォルダ振り分けの推奨アクションも同 README §2〜§4 に記載。
- 決算スケジュール上のマイルストーンは `経営管理/01_経営戦略/2026-05-12_短期生存戦略90日.md`（フェーズ4：7/10〜8/10 で決算確定・税理士へドラフト提出）を参照。
- 個人情報を含むため、取り扱いは `経営管理/README.md` の運用ルールに従う。
