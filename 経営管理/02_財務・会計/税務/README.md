# 税務 — 不足資料回収トラッカー

第 8 期決算（決算月 2026/7 末 → 8 月末申告）に向けた、税理士提出書類の裏付け資料（契約書・請求書等）の**回収進捗トラッカー**です。

## 目的

`不足資料調査/`（マーシャルアーツ様＿不足資料リスト、2026-05-27 時点）で棚卸しした不足資料の回収ステータスを一元管理し、決算確定までの残タスクを可視化する。**全件版（取引明細レベル）を master、優先版（高額サマリー）を抜粋ビュー**として併用する。

## ファイル

| ファイル | 内容 | 行数 | 列数 |
| --- | --- | ---: | ---: |
| `2026-06-01_不足資料回収トラッカー_全件.csv` | **全件 master**。入出金の取引明細単位（入金 118 / 出金 57）。 | 175 | 14 |
| `2026-06-01_不足資料回収トラッカー.csv` | 高額・優先案件 33 件（入金 19 / 出金 14）の抜粋サマリー。 | 33 | 13 |

> **データ出所:** `不足資料調査/00_不足資料リスト_raw.txt`（Excel 原本パース済）および同 `README.md` §5。**データ基準日 2026-05-27**、トラッカー作成日 2026-06-01。
> 和暦は西暦に変換（令和7年=2025 / 令和8年=2026）。全件版は原本の取引明細をそのまま 1 行 = 1 取引で展開し、高額・優先 33 件には既知の Drive/Slack 状況・回収ステータスを反映、それ以外は `未調査`／`未着手` を初期値とする。

## 列定義

### 全件版（14 列）

| 列 | 列名 | 内容 |
| --- | --- | --- |
| 1 | `区分` | `入金` / `出金` |
| 2 | `優先度` | 金額ベース（`高`=1,000 万円以上 / `中`=100 万円以上 / `低`=100 万円未満。返金等のマイナスは絶対値で判定） |
| 3 | `対象日` | 入金日 / 出金日（西暦） |
| 4 | `口座` | 入出金口座（原本どおり） |
| 5 | `取引先` | 取引先名（入金は `普通預金：` 接頭辞を除去、出金は振込先の銀行・支店・名義） |
| 6 | `金額` | 金額（円）。返金はマイナス |
| 7 | `不足種別` | `請求書不足` / `契約書不足` / `固定資産税精算金明細不足` / `売上計上先不明` / `要確認` |
| 8 | `Drive状況` | `あり` / `なし` / `未調査`（優先33件のみ調査済） |
| 9 | `Slack状況` | `あり` / `なし` / `未確認` / `-` / `確認できず` / `未調査` |
| 10 | `回収ステータス` | 下記凡例参照（優先33件以外は `未着手`） |
| 11 | `次アクション` | 次に取るべき作業 |
| 12 | `担当` | 担当者（未割当） |
| 13 | `期限` | 対応期限（未設定） |
| 14 | `備考` | 補足・疑問点（返金理由・差額など） |

### 優先版（13 列）

`全件版` から `対象日`・`口座` を除き、`取引先` 単位に集約した抜粋。列は `区分 / 優先度 / 取引先 / 対象日 / 金額 / 不足種別 / Drive状況 / Slack状況 / 回収ステータス / 次アクション / 担当 / 期限 / 備考`（全件 `優先度=高`、複数取引は `対象日=多数`・`金額=概算`）。

## 回収ステータス凡例

| ステータス | 意味 |
| --- | --- |
| `回収済` | 必要資料が Drive 内に揃っている |
| `要確認` | 資料はあるが金額・物件・対応関係のギャップ確認が必要 |
| `移行待ち` | Slack 上にあり、Drive への移行が必要（ファイル ID は次アクション欄に記載） |
| `要取得` | Drive/Slack いずれにも未発見。先方等から別途取得が必要 |

## 運用

- 進捗があった行の `回収ステータス` / `担当` / `期限` / `備考` を更新する。原則 `全件版` を master として更新し、優先版は高額案件の報告用ビューとして扱う。
- 重複削除・フォルダ振り分けの推奨アクションは `不足資料調査/README.md` §2〜§4 を参照。元データ（Excel 原本パース済）は同 `00_不足資料リスト_raw.txt`。
- 決算スケジュール上のマイルストーンは `経営管理/01_経営戦略/2026-05-12_短期生存戦略90日.md`（フェーズ4：7/10〜8/10 で決算確定・税理士へドラフト提出）を参照。
- 個人情報を含むため、取り扱いは `経営管理/README.md` の運用ルールに従う。
