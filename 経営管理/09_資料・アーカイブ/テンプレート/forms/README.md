# 業務フォーム集

> 作成日: 2026-05-26
> 用途: 業務マニュアル「機微情報の取り扱い」ポリシーに準拠した HTML フォーム雛形

各フォームは **単一の HTML ファイル**で完結する自己内包型。
ブラウザでそのまま開いて記入 → 送信時に **JSON が画面表示 + クリップボードに自動コピー**される。

提出方法は別途運用ルールで決定（Slack 投稿、Google Sheets 転記、Form Run 等）。

---

## 含まれる 7 つのフォーム

| ファイル | 用途 | 主担当 |
| --- | --- | --- |
| `sales-report-form.html` | 案件報告（ステージ進捗報告） | 営業担当 |
| `sales-report-form-complete.html` | 案件完了報告 | 嶺田 / 坂本 |
| `sales-report-update-form.html` | 案件ステージ更新報告 | 営業担当 |
| `daily-report-form.html` | 日報（活動数値 + ハイライト） | 全社員 |
| `apo-cancel-form.html` | アポキャンセル報告 | 営業担当 |
| `customer-feedback-form.html` | 顧客フィードバック（アフターフォロー） | 上田 / 営業 |
| `search-customer.html` | 顧客・案件検索（モック UI） | 全社員 |

---

## 共通仕様

- **PII を持たない設計**: 顧客実名は入力欄なし、案件 ID + 顧客匿名コード（CUST-A 形式）のみ
- **案件 ID**: `DEAL-YYYY-NNNNN` 形式を入力時バリデート
- **ステージ用語**: [銀行審査状況 パイプライン運用ガイド](../../../07_業務管理/業務マニュアル/案件管理基盤/銀行審査状況パイプライン運用ガイド.md) と統一
- **出力形式**: JSON（クリップボード自動コピー）
- **ナビ**: 各フォーム下部に他フォームへのリンク

---

## 機微情報の取り扱い

- 入力後の JSON データには案件 ID・顧客コード・銀行名等が含まれる
- 提出後の保管先（Slack / Google Sheets）は **アクセス権限制限フォルダ** に限定
- メールでの送信は禁止（誤送信リスクのため）
