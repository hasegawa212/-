## マニュアル運用ルール

### バージョン定義
- **v0.1**：AI 起稿の初版
- **v1.0（暫定）**：担当者指名済・運用ベースライン。実務責任者レビュー待ち
- **v1.0（真版）**：実務責任者レビュー完了・本運用
- **v1.x / v2.x**：実運用後の改善反映

### 作成・更新
- 改定履歴を末尾に必ず追記
- 主担当・バックアップ担当を冒頭に明記（脱属人化原則 ①）
- 各マニュアル末尾の「レビュー状況」セクションのチェックボックスを実レビュー完了時に ✓

### 機微情報の取り扱い
- マニュアルには**顧客名・物件・口座・社員人事評価等の機微情報を一切含めない**
- 具体ケース・案件記録は別途 Google Drive / Notion 等で管理
- 機微情報を含むファイルは git にコミットしない

### レビュー周期
- 各マニュアルは**四半期に 1 回**レビュー（金利水準・銀行リードタイム等の数字を更新）
- 重大な失敗ケース発生時は即時更新
