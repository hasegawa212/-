## 1. このガイドの位置づけ

複数のマスタ・台帳（銀行・司法書士・業者・案件管理台帳）を**統一的に運用する親ルール**。

各個別マスタは詳細を持ち、本ガイドは：
- マスタ全体の構造図
- アクセス権限管理
- バックアップ・引継ぎルール
- 機微情報の取り扱い基準
- システム選定基準

を統括する。

---

## 2. 情報資産マスタ 全体構成

```
[案件管理台帳]（中心）
   │
   ├── [顧客実名サブシート]（アクセス制限）
   │
   ├─→ [銀行マスタ] ── 銀行 ID で紐付け
   │
   ├─→ [司法書士マスタ] ── 司法書士 ID で紐付け
   │
   ├─→ [業者マスタ] ── 業者 ID で紐付け
   │
   ├─→ [物件マスタ] ── 物件 ID で紐付け（住所等は別管理）
   │
   └─→ [紹介者マスタ] ── 紹介元案件 ID で紐付け
```

## 4. アクセス権限 4 階層

| 階層 | 範囲 | 対象者 |
| --- | --- | --- |
| L4 全権 | 全マスタ 全件 編集・閲覧・削除 | CEO + 主担当（三浦） |
| L3 編集 | 担当マスタ 編集・閲覧 | 各マスタ主担当・バックアップ |
| L2 閲覧 + 自案件編集 | 全マスタ閲覧 + 自分の担当案件のみ編集 | 営業全員・実務担当 |
| L1 閲覧 | 全マスタ閲覧のみ（機微サブシート除く） | 一般社員 |
| L0 アクセス不可 | 機微サブシート（顧客実名・連絡先等） | L0：L4 + 自案件担当のみ |

⚠️ **顧客実名・財務情報は L0 制限**（必要な人だけが見られる状態）

## 5. 機微情報の分離ルール

「公開しても構わない部分」と「機密部分」を**シート / テーブルで物理分離**する。

### 案件管理台帳の例
- メインシート：案件 ID・顧客ハンドル（A・B・C）・物件エリア・ステータス・営業担当
- **機密シート 1**：案件 ID ↔ 顧客実名 対応表
- **機密シート 2**：案件 ID ↔ 物件詳細住所
- **機密シート 3**：案件 ID ↔ 顧客財務情報（年収・借入等）

### マスタの例
- 銀行マスタ メイン：銀行名・支店・対応エリア・標準金利
- 銀行マスタ 機密：担当者個人名・携帯・直通

→ 機密シートは**L0 アクセス制限 + 監査ログ取得**

## 6. システム選定基準

### 候補
| 候補 | メリット | デメリット | 推奨度 |
| --- | --- | --- | --- |
| **CRM ソフト**（kintone, Salesforce, HubSpot 等） | アクセス制御強い、リレーション設計可、監査ログあり | 導入コスト・学習コスト | ◎（中期推奨） |
| **Notion** | 直感的、リレーション機能、コスト低 | アクセス制御が粗い | ○（短期推奨） |
| **Google Workspace + スプレッドシート** | 即開始可、慣れている | アクセス制御が弱い、リレーション弱い | △（暫定運用のみ） |
| **個人 PC ファイル / 紙** | コスト 0 | 属人化温床、検索不能 | ✗（NG） |

### 推奨ロードマップ
- **Phase 1（〜2 ヶ月）**：Google Workspace スプレッドシートで暫定運用開始
- **Phase 2（2〜6 ヶ月）**：Notion へ移行（コスト低、機能十分）
- **Phase 3（6 ヶ月〜）**：本格 CRM 導入検討（取引規模が拡大した場合）

## 7. バックアップ・冗長化

機微情報を扱う以上、データロストは事業継続リスク：

- [ ] 日次自動バックアップ（クラウド側機能利用）
- [ ] 週次 物理エクスポート（CSV / Excel）→ 別保管
- [ ] 月次 アクセス監査ログ確認
- [ ] 四半期 担当者の権限見直し（退職・異動反映）
- [ ] 年次 全データの整合性チェック

---

## 8. 退職・引継ぎ時のルール

社員の退職・異動時：

1. **即時**：退職予定者の編集権限を閲覧に降格
2. **退職前**：担当マスタの全件を後任に引継ぎ（口頭 + 書面）
3. **退職時**：個人アカウントの完全削除、IP アドレス・端末からのアクセス遮断
4. **退職後 1 週間**：全マスタの異常変更チェック

---

## 9. PII（個人情報）取扱い基準

### 顧客 PII
- 取得時：個人情報取得同意書（書面）
- 保管期間：契約終了後 5 年
- 削除：保管期間経過後、顧客リクエスト時、退会時
- 第三者提供：原則禁止、銀行・司法書士等の必要範囲のみ

### 業者・パートナー PII（個人事業主・担当者個人）
- 取得時：取引開始時に取扱い説明
- 名刺情報：通常取扱い
- 個人連絡先（携帯等）：本人許可下で取扱い、不要時削除

### 社員情報
- 取得：入社時の同意書範囲
- 保管：人事システム（業務マスタ群と分離）
- アクセス：人事部 + 経営層のみ

## 10. 監査・点検フロー

四半期に 1 回、以下を点検：

- [ ] アクセスログの異常（深夜アクセス・大量ダウンロード等）
- [ ] 権限が適切か（昇格・降格漏れ）
- [ ] バックアップが正常か（リストアテスト）
- [ ] マスタ間の整合性（案件 ID と銀行 ID の紐付け矛盾等）
- [ ] 古い案件のアーカイブ（5 年経過分）
- [ ] 機密シートのアクセス制御確認

---

## 11. インシデント対応

機微情報の漏洩・誤送信・誤公開発生時：

1. **発覚時**：即時 関係者へ報告（24 時間以内）
2. **影響範囲特定**：何が、誰に、どこまで
3. **拡散停止**：可能な限りの回収
4. **顧客通知**：法的要件に従い当事者通知
5. **再発防止**：原因究明 + マニュアル更新
6. **記録**：インシデント記録に永久保管
