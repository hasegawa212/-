# アフターフォロー マニュアル v1.0（暫定）

> 作成日: 2026-05-16
> 主担当: **上田**（営業課長・顧客接点責任者）
> バックアップ担当: **営業全員（各案件の主担当営業）**
> 派生元: [住宅ローン決済日対応マニュアル §10](../決済対応/住宅ローン決済日対応マニュアル.md)、[リフォーム工事進行管理マニュアル §9](./リフォーム工事進行管理マニュアル.md)
> 脱属人化スキーム該当原則: ②知識の文書化、③顧客は会社が保有

---

## 1. このマニュアルの目的

引渡し後のお客様フォローを**「営業担当の気分次第」から「会社の標準業務」に**変える。

アフターフォローは、収益直結業務ではないため軽視されがちだが：
- **紹介経由案件の創出源**（顧客から紹介を引き出す）
- **クレームの早期発見・予防**（小さい不満が訴訟化するのを防ぐ）
- **会社のブランド維持**（口コミ・SNS への影響）

→ 「やる/やらない」を担当者個人に委ねず、**標準フォローを全件実施**する。

---

## 2. アフターフォロー 標準スケジュール

```
[引渡し日]
   ↓ +3 日
[Step 1] 不具合確認連絡（電話 / LINE）
   ↓ +1 週間
[Step 2] 入居状況確認
   ↓ +1 ヶ月
[Step 3] 1 ヶ月点検（業者保証範囲）
   ↓ +3 ヶ月
[Step 4] 3 ヶ月フォロー連絡
   ↓ +6 ヶ月
[Step 5] 6 ヶ月フォロー連絡
   ↓ +1 年
[Step 6] 1 年点検 + 紹介依頼
   ↓ 年 1 回
[Step 7+] 年次フォロー（年賀状・近況確認）
```

---

## 3. 各ステップの詳細

### Step 1：3 日後 不具合確認

**目的**：入居直後の小さな不具合を即時把握し、業者保証で対応

**連絡内容（スクリプト）**：
```
[お客様]様、引渡しから 3 日経ちましたが、何かお気付きの点は
ございますか？

特に：
- 設備の動作（給湯器・空調・換気扇）
- 水回り（水漏れ・排水）
- 鍵・建具の動作
- 電気系統

小さなことでも構いませんので、お知らせください。
```

**対応**：
- 不具合あり → 業者へ即連絡（48 時間以内対応）
- 不具合なし → 「お困りごとがあればいつでもご連絡ください」で締め

### Step 2：1 週間後 入居状況確認

**目的**：生活が落ち着いたかどうか、追加の困りごと把握

**連絡内容**：
- 生活ライフライン（電気・ガス・水道）の手続き完了確認
- 物件規約に関する質問対応
- 近隣との関係性確認

### Step 3：1 ヶ月点検

**目的**：業者保証範囲内のチェック実施

**実施内容**：
- 業者同行で現場点検（30 分目安）
- 主要設備の動作確認
- 不具合があれば**業者保証で無償修理**
- 点検記録を案件ファイルに保管

### Step 4：3 ヶ月フォロー連絡

**目的**：季節変化対応の確認

**連絡内容**：
- 季節の挨拶
- 季節特有の使い方アドバイス（冬：給湯器凍結防止 / 夏：空調メンテ等）
- 困りごと有無の確認

### Step 5：6 ヶ月フォロー連絡

**目的**：関係性の維持 + 半年点検アナウンス

**連絡内容**：
- 半年経過の御礼
- 紹介可能なお知り合いの有無を**さりげなく**確認
- 次回 1 年点検の予告

### Step 6：1 年点検 + 紹介依頼

**目的**：本格的な紹介依頼の場

**実施内容**：
- 1 年点検（現場 or オンライン）
- お客様満足度ヒアリング（NPS スコア取得）
- **紹介依頼**（同僚・親族・友人で住宅購入予定者の有無）
- 紹介報酬制度の説明（あれば）

### Step 7+：年次フォロー

**目的**：長期関係の維持

**内容**：
- 年賀状送付（年 1 回 必須）
- 引越し記念日連絡（任意）
- 大きな市況変動時の情報提供（金利変更等）

## 4. 案件ファイルへの記録

各ステップで以下を記録：

- [ ] 連絡日時
- [ ] 連絡手段（電話 / メール / LINE / 訪問）
- [ ] 担当者
- [ ] お客様の反応・コメント
- [ ] 検出された不具合（あれば）
- [ ] 業者対応の有無・結果
- [ ] 紹介情報（あれば）
- [ ] 次回連絡予定

→ **記録なし = 実施なし** と判断する運用ルール。

---

## 5. クレーム発生時の対応

アフターフォロー中にクレーム発生時：

| クレーム種別 | 一次対応 | エスカレーション |
| --- | --- | --- |
| 軽微な不具合 | 業者修理対応 | 24 時間以内 |
| 設備故障 | 業者保証 or メーカー保証確認 | 48 時間以内 |
| リフォーム品質問題 | 業者責任で再施工 | 1 週間以内 |
| 契約・説明不足クレーム | 営業担当 + 上田 同席 | 即日 |
| 第三者経由クレーム | 経営層 No.2 / CEO へ報告 | 即日 |
| SNS 投稿等の公開クレーム | 経営層判断 | 即時 |

→ **小さなクレームを軽視すると訴訟化**。芽の段階で潰す。

## 6. 紹介経由案件の管理

紹介案件は通常案件より**成約率が高く、粗利も大きい**。

### 紹介発生時の標準フロー
1. 紹介者に御礼連絡（24 時間以内）
2. 紹介先への初回接触（紹介者経由 or 直接）
3. 案件 ID 採番（[案件 ID 採番ルール](../案件管理基盤/案件ID採番・管理台帳ルール.md)）
4. 紹介者欄に紹介元の案件 ID を記録
5. 成約時：紹介者への御礼（手紙 + 紹介報酬 if 制度あり）

### 紹介経由案件の KPI
- 月間紹介発生数（目標：5 件）
- 紹介経由の成約率（目標：60%）
- 紹介者あたりの平均紹介数（目標：2 件以上）

| ... | ... | ... | ... | ... |

→ **月次仕入れ会議の最後 5 分で確認**。未実施が多い担当には個別フォロー。

---

## 8. 引退・解約時の対応

お客様から「フォロー連絡不要」の申し出があった場合：

- 即時 連絡停止（しつこいフォローは逆効果）
- 「引退」ステータスを案件ファイルに記録
- 年賀状のみ継続（拒否があれば停止）

→ **無理に連絡を続けない**ことも重要。

---

## 9. アフターフォロー NG 集

| ❌ | 理由 |
| --- | --- |
| 売込み目的の連絡 | お客様は察知する。信頼喪失 |
| 紹介依頼の押付け | 関係性悪化 |
| 不具合連絡への遅延対応 | クレーム拡大 |
| 担当者依存（他の人が状況知らない） | 担当退職時に関係性ロスト |
| 記録なしのフォロー | 実施有無の管理不能 |
| 全員一律のテンプレ連絡 | お客様ごとの個別配慮なし |
